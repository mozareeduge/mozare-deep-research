# System decision record

## D-001 — Repository state replaces conversational memory

Claude sessions are disposable. All consequential sources, queries, evidence, claims, decisions, audits, and handoffs are stored in validated files so that clean sessions do not lose research state.

## D-002 — Methodology is compiled, not injected

The rich human methodology remains documentation. A compact integrity kernel, phase-specific skills, schemas, validators, permissions, and hooks carry its operational protections without pushing theoretical vocabulary into every model call.

## D-003 — Evidence promotion is fail-closed

A retrieval hit cannot become accepted evidence until its source exists, its exact text is validated in the recorded file, its local context is reviewed, and its access/use status permits the intended claim.

## D-004 — Candidate links remain distinct from claims

Possible analogies, shared operations, contradictions, and model suggestions are stored as candidate links. They can be tested, promoted, deferred, or rejected without requiring the system to produce a connection.

## D-005 — Retrieval precedes model interpretation

Identifier handling, caching, deduplication, lexical/vector retrieval, quotation matching, referential checks, and accounting use deterministic or local tools. Claude receives only bounded packets that require interpretation.

## D-006 — Zotero is read-only during research

The normal Claude profile can inspect the library but cannot mutate it. Library writes require a separate maintenance task, preview, explicit approval, and verification.

## D-007 — PaperQA is retrieval-only by default

PaperQA parsing and indexes may improve local corpus retrieval. Its answer agent, per-passage LLM summaries, metadata inference, and generated media enrichment are outside the default subscription-conscious workflow.

## D-008 — Sonnet with explicit effort routing

The project starts on the current `sonnet` alias at medium effort. Skills use low effort for bounded transformations, medium for evidence/claims/writing, and high for one independent central audit. Expensive escalation receives a compact packet and human approval.

## D-009 — Stop hook checks validity, not phase completion

Every ordinary Claude turn may end when repository invariants hold. A complete handoff is required only when the user invokes `/130-handoff` or runs phase-stop verification. This avoids an infinite block on normal interaction while preserving clean-session discipline.

## D-010 — Releases preserve omissions and limits

A public output may be selective, but its supporting release package records source scope, claims, challenges, limitations, omitted material, and deferred routes. Selection does not erase research history.
